# 数据定义说明 · 配置驱动原则

> 工序、字段、产品模板**全部通过 JSON 配置表达**，运行时记录只引用 `step_id` / `field_id`。  
> 应对工艺变更时：**改配置，不改代码、不改 schema**。

---

## 一、两层结构

| 层级 | 作用 | 智造数据台 |
|------|------|------------|
| **配置层** | 定义「有什么工序、采什么字段、怎么展示」 | `config-model.json` + `presets/jqhc-manufacturing-config.json` |
| **运行时层** | 存储实际采集数据 | `telemetry-record.json` + `runtime-analytics.json` |

---

## 二、智造数据台 · 配置模型

### 核心对象

```
DomainConfig
├── process_steps[]      工序目录（可增删改）
├── fields[]             全局字段/数据点目录
├── step_field_bindings[]  工序 ↔ 字段 多对多
├── product_templates[]  产品模板（工序序列）
├── kpi_definitions[]    KPI 与字段关联
└── equipment_types[]    设备类型目录
```

### 运行时记录

```json
{
  "record_type": "process_snapshot",
  "config_id": "jqhc-manufacturing",
  "process_step_id": "filling_forming",
  "equipment_id": "FILLER-01",
  "product_batch": "FCW-20260605-B2",
  "timestamp": "2026-06-05T10:15:00Z",
  "values": {
    "fill_rate_kg_h": 85.2,
    "forming_pressure_MPa": 8.5,
    "fill_ratio_pct": 18.5,
    "powder_leak_level": "slight"
  }
}
```

`values` 的键必须出现在当前 `process_step_id` 的 `step_field_bindings` 中。

### 默认配置规模（`jqhc-manufacturing`）

| 项 | 数量 |
|----|------|
| 工序 | 13 |
| 字段 | 61 |
| 工序-字段绑定 | 126 |
| 产品模板 | 3（焊条 / 实心焊丝 / 药芯焊丝） |

### 扩展方式

1. 在 `process_steps` 增加新工序（新 `step_id`）
2. 在 `fields` 增加新字段（新 `field_id`）
3. 在 `step_field_bindings` 关联工序与字段
4. 在 `product_templates` 调整工序序列
5. 产线实例（`ProductLine`）引用 `template_id` + `config_id`

## 四、应变场景示例

| 场景 | 操作 |
|------|------|
| 药芯线新增「在线检重」字段 | `fields` 加 `fill_weight_online_kg` → `step_field_bindings` 挂到 `filling_forming` |
| 新增「酸洗」工序 | `process_steps` 加 `pickling` → 建绑定 → 更新相关 `product_templates.steps` |
| 新产品「不锈钢焊丝」 | 新建 `product_template` + 复用/新增工序与字段 |
| 字段改中文名 | 改 `display_name`，`field_id` 不变 |
