# 金桥焊材 · 智造数据台原始数据源需求清单

> **文档目的**：梳理 POC 阶段 `line-simulator` 模拟的全部数据，对照 `dataplatform-api` + `dataplatform-web` 现有功能，明确项目实施时需从产线及其他业务系统采集接入的**原始数据源**，以保证软件功能完整落地。  
> **依据文件**：`jqhc-factory-master-data.json`、`jqhc-manufacturing-config.json`、`ingest-api.json`、`runtime-analytics.json`、`entities.json`  
> **版本**：2026-06-06 · 对应 POC config v1.0.0

---

## 1. 总览

| 报警 | 模拟器场景注入 + PLC | 设备 PLC/DCS + MES | 事件推送 |

## 2. 按来源系统分类的原始数据清单

### 2.1 MES（制造执行系统）— 核心编排数据源

**职责**：批次生命周期、工序流转、产线启停、WIP 跟踪、与订单/配方绑定。

| 数据对象 | 关键字段 | 对应 ingest 通道 | 支撑功能 |
|----------|----------|------------------|----------|
| 生产批次 | `batch_id`, `factory_id`, `workshop_id`, `product_line_id`, `production_order_id`, `product_category`, `recipe_id`, `grade`, `shift`, `status`, `started_at`, `ended_at`, `quantity_kg`, `parent_batches[]` | `POST /v1/ingest/batches` | 批次中心、批次详情、产线总览、订单交付、追溯 |
| 批次创建/关闭 | `event_type=BATCH_CREATE/BATCH_CLOSE`, `material_batch`, `timestamp`, `product_line_id`, `production_order_id`, `quantity_kg` | `POST /v1/ingest/events` | 批次时间线、追溯链 |
| 工序启停 | `event_type=LINE_ON/LINE_OFF`, `process_step_id`, `product_line_id`, `material_batch` | events | 工序管道状态、OEE 可用率 |
| 工序间转移 | `event_type=TRANSFER`, `from_location`, `to_location`, `process_step_id`, `quantity_kg` | events | 批次时间线、WIP 跟踪 |
| 订单下达 | `event_type=ORDER_RELEASE`, `production_order_id`, `assigned_line_ids` | events | 订单中心、Release 窗口 |
| 原料领用 | `event_type=STOCK_OUT`, `material_batch`, `location`, `quantity_kg` | events | 批次追溯、缺料告警 |
| 成品入库 | `event_type=STOCK_IN`, `finished_batch_id`, `location`, `quantity_kg` | events | 订单完成量、库存 |
| 缺料事件 | `event_type=MATERIAL_SHORTAGE`, `remark` | events | 工厂驾驶舱阻塞项 |
| 班次 | `shift`（day/night/middle） | 遥测信封 + 批次 | KPI 分班次统计 |

**采集要求**：
- 批次状态变更须实时推送（< 30s 延迟）
- `parent_batches` 必须完整，否则追溯页出现 `trace_gap`
- 批次与当前工序、设备、订单三元组须一致

### 2.2 PLC / SCADA / 边缘网关 — 过程遥测数据源

**职责**：设备运行参数、状态、能耗的实时采集，是产线工作台、趋势分析、OEE、参数合规判断的基础。

**接入协议**（主数据 `equipment.connection` 已定义）：`modbus_tcp`、`modbus_rtu`、`opc_ua`、`mqtt`

**推送格式**：`POST /v1/ingest/telemetry`

```json
{
  "source": "edge-gateway",
  "source_instance": "gw-fcw-line-07",
  "sent_at": "2026-06-06T10:00:00Z",
  "records": [{
    "record_type": "process_snapshot",
    "config_id": "jqhc-manufacturing",
    "template_id": "flux_core_wire",
    "timestamp": "2026-06-06T10:00:00Z",
    "product_line_id": "FCW-LINE-07",
    "equipment_id": "FILLER-01",
    "process_step_id": "filling_forming",
    "product_batch": "FCW-20250606-B3",
    "recipe_id": "RCP-FCW-HY830-V2",
    "shift": "day",
    "values": {
      "status": "RUNNING",
      "fill_ratio_pct": 28.5,
      "forming_pressure_MPa": 1.82,
      "fill_rate_kg_h": 420,
      "power_kw": 35.2

    "value_meta": {
      "fill_ratio_pct": { "data_source": "plc", "quality": "good", "recipe_target": 28.0 }
    }
  }]
}
```

#### 2.2.1 全工序遥测字段清单

> 字段必须在 `jqhc-manufacturing-config.json` 的 `step_field_bindings` 中注册；额外白名单：`power_kw`（能耗）、`status`、`run_mode`、`alarm_code`。

| 工序 ID | 中文名 | 适用产品 | 必采字段 (required) | 选采字段 |
|---------|--------|----------|---------------------|----------|
| `wire_drawing` | 拔丝 | 焊条 | `status`, `alarm_code`, `line_speed_m_per_min`, `outlet_diameter_um` | `run_mode`, `motor_rpm`, `current_A`, `voltage_V`, `temperature_C`, `inlet_diameter_um`, `tension_kN` |
| `cutting` | 切丝 | 焊条 | `status`, `alarm_code`, `length_mm`, `fault_code` | `run_mode`, `cutting_speed_cuts_per_min`, `total_count` |
| `powder_mixing` | 配粉搅拌 | 焊条/药芯 | `status`, `alarm_code`, `batch_id`, `recipe_id` | `run_mode`, `motor_rpm`, `motor_current_A`, `mixing_time_s`, `mixing_rpm`, `ingredients`, `ingredient_batch_ids`, `mixing_uniformity`, `mixing_uniformity_method` |
| `coating` | 压涂 | 焊条 | `status`, `alarm_code`, `coating_thickness_mm` | `run_mode`, `motor_rpm`, `current_A`, `temperature_C`, `core_speed_m_per_min`, `coating_pressure_MPa`, `coating_feed_rate` |
| `drying` | 烘干 | 焊条 | `status`, `alarm_code`, `moisture_pct` | `run_mode`, `zone_id`, `set_temp_C`, `actual_temp_C`, `belt_speed_m_per_min`, `humidity_pct`, `zone_temp_C`, `temp_curve` |
| `cut_strip` | 裁带 | 药芯 | `status`, `alarm_code`, `line_speed_m_per_min`, `strip_thickness_mm` | `run_mode`, `strip_batch_id`, `uncoil_tension_N`, `cut_length_mm` |
| `filling_forming` | 填充成型 | 药芯 | `status`, `alarm_code`, `fill_ratio_pct`, `forming_pressure_MPa` | `run_mode`, `motor_rpm`, `current_A`, `fill_rate_kg_h`, `strip_tension_N`, `powder_leak_level` |
| `rough_drawing` | 粗拔 | 实心/药芯 | `status`, `alarm_code`, `line_speed_m_per_min`, `outlet_diameter_um` | `run_mode`, `motor_rpm`, `current_A`, `voltage_V`, `temperature_C`, `inlet_diameter_um`, `tension_kN`, `surface_defect` |
| `fine_drawing` | 细拔 | 实心/药芯 | 同粗拔 | 同粗拔 |
| `copper_plating` | 镀铜 | 实心/药芯 | `status`, `alarm_code`, `coating_thickness_top_um`, `coating_thickness_bottom_um` | `run_mode`, `motor_rpm`, `current_A`, `voltage_V`, `temperature_C`, `line_speed_m_per_min`, `current_density_A_per_dm2`, `bath_temp_C`, `bath_ph`, `coating_thickness_left_um`, `coating_thickness_right_um` |
| `winding` | 层绕 | 实心/药芯 | `status`, `alarm_code`, `tension_N` | `run_mode`, `motor_rpm`, `spool_id` |
| `packaging` | 包装 | 全品类 | `status`, `alarm_code`, `spool_count` | `run_mode`, `package_count`, `finished_batch_id`, `label_content` |
| `stock_in_out` | 出入库 | 全品类 | `status`, `quantity_kg`, `location` | `agv_id`, `from_location`, `to_location`, `finished_batch_id` |
| **全设备通用** | 能耗 | 全品类 | `power_kw`（绑定 `energy_meter_id`） | — |

#### 2.2.2 前端关键监控字段（CRITICAL_FIELDS）

趋势页、产线孪生、参数表优先展示，**建议采集频率 ≥ 0.2Hz（5s 一次）**：

| field_id | 中文 | 单位 | 典型工序 |
|----------|------|------|----------|
| `fill_ratio_pct` | 填充率 | % | filling_forming |
| `forming_pressure_MPa` | 成型压力 | MPa | filling_forming |
| `coating_thickness_top_um` | 镀层厚度(上) | μm | copper_plating |
| `coating_thickness_bottom_um` | 镀层厚度(下) | μm | copper_plating |
| `tension_kN` | 张力 | kN | rough/fine_drawing |
| `line_speed_m_per_min` | 线速度 | m/min | 拔丝/裁带/拉拔 |
| `mixing_rpm` | 搅拌转速 | rpm | powder_mixing |
| `motor_rpm` | 电机转速 | rpm | 层绕等 |

#### 2.2.3 数据点元数据（主数据预置，采集时须对齐）

每个 `(equipment_id, field_id)` 在主数据 `data_points` 中登记：

| 元数据字段 | 用途 |
|------------|------|
| `data_point_id` | 全局唯一标识 |
| `display_name` | UI 显示名 |
| `data_type` / `unit` | 类型与单位 |
| `sampling_freq_hz` | 期望采样频率 |
| `priority` | critical/high/medium/low |
| `mandatory_flag` | 是否必采 |
| `spec_limits`（lsl/usl/target） | 参数合规判断、趋势图规格带 |
| `kpi_tags` | OEE/质量/能耗等标签 |
| `process_step_id` | 工序归属校验 |

**采集要求**：
- 时间戳须 UTC ISO8601，时钟偏差 < 30s（否则 `CLOCK_SKEW` 拒绝）
- 每条 record 须携带当前 `product_batch` 和 `recipe_id`（MES 提供）
- 非当前工序设备可降频，但至少推送 `status` + `power_kw`

### 2.3 APS / ERP — 订单与计划数据源

| 数据对象 | 关键字段 | 接入方式 | 支撑功能 |
|----------|----------|----------|----------|
| 生产订单 | `production_order_id`, `customer_order_id`, `factory_id`, `product_id`, `product_category`, `grade`, `recipe_id`, `assigned_line_ids[]`, `priority`, `planned_quantity_t`, `released_quantity_t`, `due_date`, `status`, `order_type`, `delivery_sla_days`, `customer_segment`, `is_export` | `GET/POST/PATCH /v1/meta/orders` 或 ERP 同步 | 订单中心、订单详情、交付风险、工厂 KPI |
| 客户订单关联 | `customer_order_id` | ERP | 出口/定制订单标识 |
| 产能计划 | `design_capacity_t_per_year`, `planned_shift_pattern` | 主数据 | 产能利用率、订单排产 |

**订单状态枚举**：`draft` → `released` → `in_progress` → `completed` / `cancelled` / `on_hold`

**采集要求**：
- 订单创建/变更/完工须同步（支撑 Release 窗口与风险计算）
- `assigned_line_ids` 必须与产线主数据一致
- `due_date` + `delivery_sla_days` 用于交付风险 `days_to_due`

### 2.4 WMS / AGV 调度 — 仓储物流数据源

| 数据对象 | 关键字段 | 接入通道 | 支撑功能 |
|----------|----------|----------|----------|
| 库存快照 | `inventory_class`(raw/wip/finished), `sku`, `batch_id`, `quantity_kg`, `location`, `status` | 主数据初始化 + WMS 周期同步 | 批次中心库存 Tab |
| 原料批次 | `material_batches[]`（批次号、供应商、入库日期） | 主数据 / WMS | 配方追溯、缺料判断 |
| 物料事件 | `STOCK_IN`, `STOCK_OUT`, `TRANSFER` | events | 物料事件流、追溯 |
| AGV 事件 | `AGV_DISPATCH`, `AGV_ARRIVE`, `agv_id`, `from_location`, `to_location` | events | 批次时间线、物流完成率 |
| 物流任务 | `task_id`, `task_type`, `material_batch`, `source_location_id`, `target_location_id`, `status`, `agv_id`, `created_at`, `completed_at`, `failure_reason` | events（`LOGISTICS_TASK`） | 工厂驾驶舱、订单阻塞归因 |

**库位编码示例**：`WH-RAW`（原料仓）、`WH-WIP`（在制品）、`WH-FG`（成品仓）、产线工序位 `FCW-LINE-07/filling_forming`

**物流任务类型**：`feed_line`, `transfer_wip`, `stock_in`, `stock_out`, `return_to_hold`, `replenishment`

### 2.5 QMS / LIMS — 质量数据源

| 数据对象 | 关键字段 | 接入通道 | 支撑功能 |
|----------|----------|----------|----------|
| 过程质量门 | `gate_event_id`, `batch_id`, `process_step_id`, `gate_type`(incoming/in_process/pre_transfer/final_release), `decision`(pass/hold/rework/scrap), `reason_text`, `decided_at` | events（`QUALITY_GATE`） | 工厂驾驶舱、批次详情、订单阻塞 |
| 力学终检 | `rm_mpa`, `impact_kv2_j`, `elongation_pct`, `corrosion_i_value`, `mechanical_pass`, `grade` | events（`QUALITY_LAB`） | 质量合格率、批次质检结果 |
| 产品力学标准 | `mechanical_spec`（`rm_mpa_min`, `impact_kv2_j_min`, `elongation_pct_min`, `corrosion_i_value_min`） | 主数据 `products` | 自动判定合格/不合格 |
| 在线工艺质检 | `mixing_uniformity`, `surface_defect`, `powder_leak_level`, `coating_thickness_*` | 遥测（已含在 PLC 采集） | 过程 SPC、趋势告警 |

**质量门决策**直接影响：订单 `quality_blocked`、批次 `hold` 状态、工厂 KPI `hold_batches`

### 2.6 EMS / 能源管理系统 — 能碳数据源

| 数据对象 | 关键字段 | 来源 | 支撑功能 |
|----------|----------|------|----------|
| 设备实时功率 | `power_kw`（每设备/电表） | PLC 电表 / EMS | 产线能耗、全厂能源看板 |
| 能源资产 | `asset_id`, `asset_type`(pv/wind/grid/waste_heat), `rated_power_kw`, `annual_generation_kwh` | 主数据 + EMS | 绿电占比、年发电量 |
| 绿电实时曲线 | `hourly_green_curve[]`（`hour`, `green_power_ratio_pct`） | 光伏/风电监测 | 能碳决策 24h 曲线 |
| 碳排放因子 | 电网排放系数、绿电抵扣系数 | 主数据 `simulation_defaults` / 能碳配置 | 碳足迹、Scope2 核算 |
| 批次能耗 | 批次维度的 `consumption_kwh`, `carbon_intensity_kg_per_t` | MES 产量 + EMS 计量 | 批次碳足迹 Tab |

**POC 说明**：当前绿电占比由主数据默认比例 + 遥测 `power_kw` 汇总**计算**得出；真实实施需接入 EMS 实时绿电/市电分表计量。

### 2.8 主数据 / 设备台账 / 数字孪生 — 静态配置数据源

## 4. 接入规范摘要

### 4.1 API 端点

| 方法 | 路径 | 数据类型 | 频率建议 |
|------|------|----------|----------|
| POST | `/v1/ingest/telemetry` | 过程遥测 | 2–10s（关键参数 5s） |
| POST | `/v1/ingest/events` | 物料/报警/质门/物流事件 | 事件驱动，实时 |
| POST | `/v1/ingest/batches` | 批次状态变更 | 事件驱动 |
| POST | `/v1/ingest/heartbeat` | 网关心跳 | 10–30s |
| — | `/v1/meta/*` | 主数据查询/订单 CRUD | 按需 / 日同步 |

### 4.2 数据质量要求

| 规则 | 说明 |
|------|------|
| 字段白名单 | 遥测 `values` 的 key 必须在域配置 `step_field_bindings` 中 |
| 设备/工序校验 | `equipment_id` 须存在于主数据且 `process_step_id` 匹配 |
| 时间同步 | NTP 对齐，偏差 > 30s 拒绝 |
| 批次一致性 | 同一产线同一时刻仅一个 `in_progress` 批次 |
| ID 全局唯一 | 所有 `*_id` 字段全厂不重复 |
| 孪生一致性 | `twin_layouts.equipment_id` == `equipment.equipment_id` == 遥测 `equipment_id` |

### 4.3 建议采集频率

| 优先级 | 字段类型 | 频率 |
|--------|----------|------|
| critical | status, alarm_code, CRITICAL_FIELDS | ≥ 0.2 Hz（5s） |
| high | 其他过程参数 | ≥ 0.1 Hz（10s） |
| medium | 非活跃工序设备 | ≥ 0.05 Hz（20s），至少 status + power_kw |
| 事件类 | 批次/物料/报警/质门 | 实时（< 5s 延迟） |
| 主数据 | 设备/产线/配方变更 | 日同步或变更触发 |
