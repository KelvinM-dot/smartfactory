# 基线 · 数据结构（v1.0）

> **状态**：最终基线，后续开发与设计以此为准。  
> **原则**：工序与字段通过 JSON 配置定义；运行时仅用 `step_id` + `field_id` + `values`。  
> **客户基线**：金桥焊材，无 ERP/MES/PLM/WMS；OT 先行，自研产线 + 局部自动化（AGV、智能配方等）。

---

## 一、架构：配置层 + 运行时层

| 层级 | 智造数据台（PC） | 现场采集中台（移动） |
|------|------------------|----------------------|
| 配置 schema | `schemas/智造数据台/config-model.json` | `schemas/现场采集中台/form-config-model.json` |
| **默认配置（权威）** | `schemas/智造数据台/presets/jqhc-manufacturing-config.json` | `schemas/现场采集中台/presets/jqhc-survey-config.json` |
| 运行时 schema | `telemetry-record.json` + `runtime-analytics.json` | `observation-record.json` + `entities.json` |
| 枚举 | `enums.json` | `enums.json` |
| 示例 | `examples.json` | `examples.json` |

两产品 **完全独立**，无运行时耦合。调研字段可通过 `maps_to_manufacturing_field_id` 与制造域字段 ID 对照。

## 二、智造数据台 · DomainConfig

### 配置标识

- `config_id`: `jqhc-manufacturing`
- `config_version`: `1.0.0`

### 规模（v1.0）

| 项 | 数量 |
|----|------|
| 工序 `process_steps` | 13 |
| 字段 `fields` | 63（含 mixing_uniformity 双算法） |
| 绑定 `step_field_bindings` | 128 |
| 产品模板 `product_templates` | 3 |

### 产品模板与工序链

**焊条 `welding_rod`**
```
wire_drawing → cutting → powder_mixing → coating → drying → packaging → stock_in_out
```

**实心焊丝 `solid_wire`**
```
rough_drawing → fine_drawing → copper_plating → winding → packaging → stock_in_out
```

**药芯焊丝 `flux_core_wire`**
```
cut_strip → powder_mixing → filling_forming → rough_drawing → fine_drawing → copper_plating → winding → packaging → stock_in_out
```

### 工序目录

| step_id | 名称 | 适用产品 |
|---------|------|----------|
| wire_drawing | 拔丝 | 焊条 |
| cutting | 切丝 | 焊条 |
| powder_mixing | 配粉搅拌 | 焊条、药芯 |
| coating | 压涂 | 焊条 |
| drying | 烘干 | 焊条 |
| cut_strip | 裁带 | 药芯 |
| filling_forming | 填充成型 | 药芯 |
| rough_drawing | 粗拔 | 实心、药芯 |
| fine_drawing | 细拔 | 实心、药芯 |
| copper_plating | 镀铜 | 实心、药芯 |
| winding | 层绕 | 实心、药芯 |
| packaging | 包装 | 全部 |
| stock_in_out | 出入库 | 全部 |

### 数据三大类

| data_category | 说明 | 典型 field_id |
|---------------|------|---------------|
| equipment_status | 设备状态 | status, run_mode, motor_rpm, current_A, alarm_code |
| process_parameter | 工艺参数 | fill_ratio_pct, coating_thickness_top_um, recipe_id |
| material_flow | 物料流转 | quantity_kg, location, agv_id, finished_batch_id |

### 高价值字段（红标）

| 工序 | field_id | 关联 KPI |
|------|----------|----------|
| filling_forming | fill_ratio_pct | 一次合格率、Rm |
| copper_plating | coating_thickness_*_um | 耐蚀 I 值 |
| powder_mixing | ingredients, ingredient_batch_ids | Rm/KV₂ 追溯 |
| powder_mixing | mixing_uniformity + mixing_uniformity_method | 混合均匀性（双算法可选） |
| drying | moisture_pct | KV₂ |

### 混合均匀度双算法（v1.0 决议）

| mixing_uniformity_method | 说明 |
|--------------------------|------|
| `current_fluctuation` | 搅拌电流波动分析 → `mixing_uniformity` 分值 |
| `nir_probe` | 在线近红外探头 → `mixing_uniformity` 分值 |

两算法均为**可选、非必采**；记录时须同时填写 `method` + `value`。

### 其他运行时实体

| 实体 | schema 文件 | 用途 |
|------|-------------|------|
| MaterialEvent | runtime-analytics.json | 上下线、出入库、AGV |
| AlarmEvent | runtime-analytics.json | 报警 |

### 命名规范


### 调研模板与工序链

**药芯焊丝 `flux_core_wire`**（粗拔/细拔已拆分，与制造域对齐）
```
cut_strip → powder_mixing → filling_forming → rough_drawing → fine_drawing → copper_plating → winding_packaging
```

**焊条 `welding_rod`**
```
wire_drawing → cutting → powder_mixing → coating → drying → packaging_stock_in
```

**实心焊丝 `solid_wire`**（包装含 AGV/库存观察）
```
rough_drawing → fine_drawing → copper_plating → winding → packaging
```

### 包装相关工序说明

| step_id | 用途 |
|---------|------|
| winding_packaging | 药芯：层绕 + 包装 + 入库（一体观察） |
| packaging_stock_in | 焊条：包装 + 入库（无层绕） |
| packaging | 实心焊丝：包装 + AGV/库存/标签（已扩展 11 个绑定字段） |

### 运行时观察记录

```json
{
  "observation_id": "OBS-001",
  "task_id": "SUR-20260605-001",
  "config_id": "jqhc-survey",
  "template_id": "flux_core_wire",
  "record_scope": "process_step",
  "process_step_id": "filling_forming",
  "field_values": {
    "fill_rate_kg_h": { "value": 85.2, "source": "hmi_manual", "collectability": "plc_available" },
    "need_online_detection": "yes"
  },
  "field_annotations": {
    "fill_rate_method": { "value_priority": "red", "kpi_tags": ["quality"] }
  }
}
```

### 价值标注

| value_priority | 含义 |
|----------------|------|
| red | 直接影响 Rm/KV₂/耐蚀 I 值 |
| yellow | 影响交期/OEE |
| green | 可替代人工记录 |

### 可采集性 collectability

`plc_available` · `hmi_display_only` · `pressure_gauge_only` · `manual_record` · `needs_sensor` · `not_available` · `barcode_scannable` · `ocr_possible`

## 四、扩展规则

1. 新增工序：在 `process_steps` 注册 `step_id`
2. 新增字段：在 `fields` / `observation_fields` 注册 `field_id`
3. 挂接字段：`step_field_bindings` 增加绑定
4. 新产品模板：在 `product_templates` / `survey_templates` 排列工序序列
5. **禁止**为每个工序新增独立 JSON schema；一律走配置
