# 基线 · 现场采集中台（v1.0）

> **产品**：客户现场蹲点调研 + 样例数据采集（移动端为主）  
> **状态**：最终需求基线  
> **数据结构**：见 [基线-数据结构.md](./基线-数据结构.md) · 配置 `schemas/现场采集中台/presets/jqhc-survey-config.json`

---

## 1. 客户与策略基线

| 项 | 说明 |
|----|------|
| 使用场景 | 驻场工程师跟班蹲点，记录可采集性、样例数据、痛点 |
| 建议蹲点时长 | ≥8 小时，覆盖白班 + 部分夜班 |
| 入场身份 | 正式授权的联合项目组成员（非隐蔽调研） |
| 首期重点模板 | 药芯焊丝 `flux_core_wire` |
| 沟通原则 | 「帮大家干得更轻松更准」，非审计式质问 |

---

## 2. 产品目标

1. 将蹲点手册结构化为可现场操作的配置化表单。
2. 勾选 + 短填，减少车间文字输入。
3. 留存照片、样例数据与可采集性结论。
4. 自动汇总高价值点、优先级矩阵、采集点建议。
5. 输出数据现状报告、试点建议书、总经理访谈记录。

---

## 3. 用户角色

系统管理员 · 调研工程师 · 项目经理 · 审核人 · 只读查看者

## 4. 调研模板与工序（v1.0 最终）

### 药芯焊丝 `flux_core_wire`

```
裁带 → 配粉搅拌 → 填充成型 → 粗拔 → 细拔 → 镀铜 → 层绕包装入库
```

> **v1.0 决议**：药芯拉拔已拆为 `rough_drawing` + `fine_drawing`，与制造域一致。原 `drawing` 工序已废弃。

### 焊条 `welding_rod`

```
拔丝 → 切丝 → 配粉搅拌 → 压涂 → 烘干 → 包装入库
```

> 使用 `packaging_stock_in`（无层绕）。

### 实心焊丝 `solid_wire`

```
粗拔 → 细拔 → 镀铜 → 层绕 → 包装
```

> **v1.0 决议**：`packaging` 已扩展 AGV 等待、库存登记、标签、销售查询等观察项（与焊条物流观察对齐）。

## 5. 观察维度

**看** 设备/HMI/纸质表 · **听** 对话/异响/报警 · **问** 关键提问 · **记** 可采集性与异常

### 价值标注

| 级别 | 含义 |
|------|------|
| red | Rm/KV₂/耐蚀 I 值 |
| yellow | 交期/OEE |
| green | 可自动化替代人工记录 |

### 可采集性

plc_available · hmi_display_only · pressure_gauge_only · manual_record · needs_sensor · not_available · barcode_scannable · ocr_possible

## 6. 功能范围（完整定义）

### 6.1 任务与记录

| 模块 | 能力 |
|------|------|
| 调研任务 | 创建/归档；日期、产线、模板、牌号、班次、记录人 |
| 班前确认 | 计划牌号、实际批次、近期质量停线 |
| 现场授权 | 拍照/录音授权状态 |
| 全局信息 | `record_scope: global_info` |
| 工序观察 | `record_scope: process_step`，按模板工序向导 |
| 跨工序观察 | `record_scope: cross_process` |
| 样例数据 | HMI/仪表/纸质抄录，`SampleDataRecord` |
| 可采集性评估 | 逐字段 collectability + 传感器建议 |
| 高价值标注 | 红/黄/绿 + KPI 标签 |
| 关键提问 | 预置问题库 + 问答记录 |
| 开放问题 | 待次日验证清单 |

### 6.2 媒体与离线

| 模块 | 能力 |
|------|------|
| 拍照 | 标签/HMI/配料单/缺陷；分类与 OCR/条码（可选） |
| 语音备注 | 短录音 + 转写（可选） |
| 离线 | 表单离线填写 ≥24h，联网同步 |

### 6.3 汇总与交付

| 模块 | 能力 |
|------|------|
| 日终汇总 | 高价值点、痛点、追溯断点 |
| 优先级矩阵 | 影响度 × 实施难度四象限 |
| 采集点清单导出 | 含 `data_point_id_suggestion`、`maps_to_manufacturing_field_id` |
| 数据现状报告 | 可采集统计、人工痛点、断点 |
| 试点范围建议书 | 覆盖工序、硬件增量、KPI 目标 |
| 总经理访谈 | 12 题（战略/运营/质量/合规）+ 5 项优先级排序 |
| 导出 | JSON / XLSX / PDF / CSV |

### 6.4 配置管理

| 模块 | 能力 |
|------|------|
| 模板配置 | 工序序列、观察字段、关键提问 |
| 字段配置 | input_type、enum、composite、默认红标 |
| 访谈题库 | `interview_questions` 可维护 |

## 7. 各工序核心观察项（摘要）

完整字段见 `jqhc-survey-config.json` 的 `step_field_bindings`。

| 工序 | 核心观察 |
|------|----------|
| cut_strip | 钢带批次/厚度、张力、长度、异常断带跑偏、传感器需求 |
| powder_mixing | 配方、原料批次、称重偏差、搅拌电流、环境温湿度、替代记录、**混合均匀度算法** |
| filling_forming | 漏粉、填充速度/压力、填充率方法、在线检测需求、配粉→填充等待时间 |
| rough_drawing / fine_drawing | 拉拔力/电流、表面缺陷、速度匹配、粉末批次追溯 |
| copper_plating | 镀液参数、厚度检测方式/频次、客户投诉、参数追溯 |
| winding_packaging | 标签、AGV、入库、库存可视化（药芯） |
| packaging_stock_in | 同上（焊条，无层绕） |
| packaging | 包装 + AGV/库存/标签（实心，已扩展） |
| cross_process | 计划执行、异常处理、交接班、批次能耗、绿电排产 |

### 配粉搅拌均匀度（双算法，可选）

观察字段 `mixing_uniformity_method`：

- `current_fluctuation` — 搅拌电流波动分析
- `nir_probe` — 在线近红外探头

对应制造域 `mixing_uniformity` + `mixing_uniformity_method`。

## 8. 总经理访谈（内置）

**战略** 高端客户要求/追溯丢单 · 多标准认证 · 焊丝产能瓶颈  

**运营** 定制单延期 · AGV 隐性等待 · 焊条满产焊丝不足  

**质量成本** 高强合格率 · 绿电与能耗 · 配方经验依赖  

**风险合规** 碳足迹/CBAM · OT 安全 · 二期数据标准统一  

**优先级排序（5 项）** 缩交期 · 提合格率 · 碳足迹 · 降停机 · 出口合规库
